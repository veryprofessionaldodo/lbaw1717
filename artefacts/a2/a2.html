2 . User Stories

2.1 User

Identifier

Name

Priority

Description

US01

Search Public Projects

High

As an User, I want to search for a name of a public project, so I may find all projects that might interest me.

US02

Visualize Public Projects

High

As an User, I want to check the description, statistics about the tasks and sprints completed, and team members’ names, so I know what I’m working on, and who I’m working with.

Name

Priority

2.4. Team Member

Identifier

Name

Priority

Description

US31

Able to see all the tasks and sprints of the project

High

As a Team Member, I want to see all the tasks and sprints completed and also the ones that still need to be completed, so that I can be informed about the current status of the project.

US32

Mark Tasks as Completed

High

As a Team Member, I want to mark a task as completed, so that I can inform the entire team that it was completed.

US33

Assign task to self

High

As a Team Member, I want to assign myself a task, so that I can be responsible for its development and conclusion.

US34

Remove assignment of task to self

High

As a Team Member, I want to remove the assignment of a task to myself because I can’t complete it or be responsible for it.

US35

Participate in the Forum

Medium

As a Team Member, I want to comment on the forum and create threads in order to discuss a particular subject/problem regarding the project.

US36

Report User

Medium

As a Team Member, I want to report users who may be unproductive to work with or hostile, so I can work more calmly and effectively.

A2. Annex: Supplementary requirements

A2.1. Business Rules

Identifier

Name

Description

BR01

Ethical Development

The project coordinators are responsible for the report of team members who actively violate Behavioural Guidelines, so System Administrators might remove them.

BR02

Software Credits

After the completion of a project, if the final product is published to the public, it is not mandatory to inform the management of the project was made using the platform.

A2.2. Technical Requirements

Identifier

Name

Description

TR01

Availability

The system must be permanently available for usage.

TR02

Accessibility

Platform must be able to be used on all browsers and operating systems. 

TR03

Database

The system for the management of the database is PostgreSQL 9.4.

TR04

Stability

On the occasion of errors, the system must be able to handle them appropriately, not resulting in any crashes.

TR05

Security

The platform will maintain privacy of the user’s information by utilizing an authentication and verification system.

TR06

Ethics

The system must respect the ethical principles in software development, such as the encryption of passwords.

TR07

Scalability

The system must be prepared to deal with the growth in the number of users and their actions

TR08

Web application

The platform should be implement as a Web application with dynamic pages, using HTML5, CSS3, Javascript and PHP.

TR09

Usability

The platform should be simple and easy to use, with easy to understand functionalities.

TR10

Performance

The system should have fast response times, shorter than 3s, so as to not lose the user’s attention.

TR11

Security and Robustness

The platform will verify every input to check for vulnerabilities and attacks.
